# Lesson 04 - POST Request

## Objective

Understand how POST requests are used to create new resources on a server.

## What is a POST Request?

A POST request is used to send data to a server for processing or creation of a new resource.

Unlike GET requests, POST requests typically contain a request body.

## Syntax

```http
POST /users
```

The request body usually contains JSON data.

Example:

```json
{
  "name": "Mohit",
  "email": "mohit@example.com"
}
```

## Example API

```text
https://jsonplaceholder.typicode.com/users
```

## Request Body

```json
{
  "name": "Mohit Waghmare",
  "username": "mohitw",
  "email": "mohit@example.com"
}
```

## Headers

```http
Content-Type: application/json
```

## Expected Response

The API returns the created resource along with an ID generated by the server.

Example:

```json
{
  "id": 11,
  "name": "Mohit Waghmare",
  "username": "mohitw",
  "email": "mohit@example.com"
}
```

## Status Codes

### 201 Created

The resource was successfully created.

### 400 Bad Request

The request data is invalid.

### 401 Unauthorized

Authentication is required.

## Common Use Cases

* Create a user
* Create an order
* Submit a form
* Create a product
* Register a new account

## My Learning

* POST requests are used to create resources.
* Data is sent in the request body.
* JSON is the most common payload format.
* Successful creation usually returns HTTP 201.
